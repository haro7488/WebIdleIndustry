# WebIdleIndustry - 게임 기획서 v1

## 컨셉

**"자본가 시뮬레이터"** — 자본을 슬라이더로 배분하고, 이벤트에 대응하며 수익률을 최적화하는 방치형 게임.

> 핵심 한 줄: "돈이 돈을 버는 구조를 설계하라"

---

## 핵심 루프

```
┌─────────────────────────────────────────────┐
│                                             │
│   자본 배분 (슬라이더)                        │
│       ↓                                     │
│   시간 경과 → 수익 발생                       │
│       ↓                                     │
│   이벤트 발생 → 선택지 제시                    │
│       ↓                                     │
│   보정치(팩터) 적용                           │
│       ↓                                     │
│   슬라이더 재조정 (최적화)                     │
│       ↓                                     │
│   자본 성장 → 새 섹터 해금                     │
│       ↓                                     │
│   (반복)                                     │
│                                             │
└─────────────────────────────────────────────┘
```

## 1. 투자 섹터 (슬라이더 항목)

슬라이더로 총 자본의 %를 배분한다. 합계는 항상 100%.

### 초기 섹터 (게임 시작 시 해금)

| 섹터 | 기본 수익률 | 리스크 | 특성 |
|------|-----------|--------|------|
| 부동산 | 낮음 (2%/s) | 매우 낮음 | 안정적, 느림 |
| 주식 | 중간 (5%/s) | 중간 | 변동성 있음 |
| 사업 | 높음 (8%/s) | 높음 | 이벤트 영향 큼 |

### 해금 섹터 (자본 성장에 따라)

| 해금 조건 | 섹터 | 기본 수익률 | 리스크 | 특성 |
|-----------|------|-----------|--------|------|
| 자본 1K | 기술투자 | 10%/s | 매우 높음 | 대박 또는 쪽박 |
| 자본 10K | 해외시장 | 6%/s | 중간 | 환율 이벤트 영향 |
| 자본 100K | 암호화폐 | 15%/s | 극한 | 변동성 극대 |
| 자본 1M | 로비/정치 | 3%/s | 낮음 | 다른 섹터 보정치 강화 |
| 자본 100M | 우주산업 | 20%/s | 극한 | 최종 콘텐츠 |

## 2. 수익 계산 공식

```
섹터별 수익 = 배분된 자본 × 기본 수익률 × 팩터 보정치 × 리스크 변동

총 수익/초 = Σ(각 섹터 수익)
```

### 리스크 변동
- 리스크가 높을수록 수익의 변동 폭이 크다
- 매 틱마다 리스크 범위 내에서 랜덤 보정

```
리스크 변동 = 1.0 + (random(-1, 1) × 리스크계수)

리스크계수:
  매우 낮음 = 0.02  (98%~102%)
  낮음     = 0.05  (95%~105%)
  중간     = 0.15  (85%~115%)
  높음     = 0.30  (70%~130%)
  매우 높음 = 0.50  (50%~150%)
  극한     = 0.80  (20%~180%)
```

## 3. 이벤트 시스템

### 이벤트 발생 규칙
- **일반 이벤트**: 30~90초마다 랜덤 발생
- **긴급 이벤트**: 특정 조건 충족 시 발생 (예: 주식에 50% 이상 배분 중일 때)
- **마일스톤 이벤트**: 자본 목표 달성 시 고정 발생

### 이벤트 예시

#### 일반 이벤트
```
📰 "부동산 시장 호황"
  → [공격적 투자] 부동산 수익률 ×2.0 (60초간)
  → [관망]        부동산 수익률 ×1.2 (120초간)

📰 "기술주 폭락 조짐"
  → [손절]   기술투자 배분을 0%로 초기화, 현재 자본 보존
  → [존버]   30% 확률로 ×3.0, 70% 확률로 ×0.3 (60초간)

📰 "신흥국 경제위기"
  → [공매도] 해외시장 수익률 반전 (손실→수익), 60초간
  → [철수]   해외시장 배분 0%로, 자본 보존
  → [저가매수] 해외시장 팩터 영구 +10%, 단 30초간 수익 없음
```

#### 긴급 이벤트
```
🚨 "금융위기 발생!"
  조건: 총 자본 50K 이상
  → 모든 섹터 수익률 ×0.5 (120초)
  → [대응 선택] 한 섹터에 긴급 올인 가능 (해당 섹터만 ×1.5)

🚨 "정부 규제 강화"
  조건: 암호화폐 배분 30% 이상
  → 암호화폐 수익률 ×0.1 (180초)
  → [로비] 자본 10% 소모, 규제 즉시 해제
```

#### 마일스톤 이벤트
```
🏆 자본 10K 달성: "첫 번째 백만장자"
  → 영구 보정치: 전체 수익률 +5%
  → 새 섹터 해금: 해외시장

🏆 자본 1M 달성: "업계의 거물"
  → 영구 보정치: 이벤트 선택지 +1개
  → 새 섹터 해금: 로비/정치
```

## 4. 팩터(보정치) 시스템

모든 보정치는 곱연산으로 적용된다.

### 팩터 종류

| 종류 | 지속 | 획득 방법 |
|------|------|----------|
| 이벤트 팩터 | 일시적 (30~180초) | 이벤트 선택지 |
| 영구 팩터 | 영구 | 마일스톤, 특정 선택 |
| 시너지 팩터 | 조건부 상시 | 섹터 배분 조합 |
| 환생 팩터 | 환생 후 영구 | 프레스티지 |

### 시너지 팩터 (배분 조합 보너스)

```
"분산투자": 4개 이상 섹터에 10%+ 배분 → 전체 수익률 ×1.1
"올인":    한 섹터에 80%+ 배분 → 해당 섹터 ×1.5, 나머지 ×0.5
"밸런스":  모든 해금 섹터가 5%+ → 리스크 변동 50% 감소
"기술+사업": 둘 다 20%+ → 두 섹터 모두 ×1.3
```

## 5. 진행 구조

### Phase 1: 초보 투자자 (자본 0 ~ 1K)
- 3개 기본 섹터
- 단순 이벤트
- 슬라이더 조작 학습

### Phase 2: 중급 투자자 (자본 1K ~ 100K)
- 5개 섹터
- 복합 이벤트 + 시너지
- 리스크 관리 학습

### Phase 3: 거물 (자본 100K ~ 10M)
- 7개 섹터
- 긴급 이벤트 + 글로벌 위기
- 최적화 심화

### Phase 4: 재벌 (자본 10M+)
- 전체 섹터 해금
- 환생(프레스티지) 가능
- "영향력" 시스템

## 6. 환생 (프레스티지) 시스템

### 조건
- 자본 10M 이상에서 "제국 재건" 선택 가능

### 리셋 항목
- 자본 → 100으로 초기화
- 섹터 해금 → 초기 3개로
- 이벤트 팩터 → 초기화

### 유지 항목
- 환생 포인트 (자본 규모에 비례)
- 영구 팩터 일부

### 환생 보너스
```
환생 포인트로 구매:
  - "선견지명": 이벤트 결과 미리보기 (확률 표시)
  - "연줄": 이벤트 선택지 +1개
  - "자본력": 기본 수익률 영구 ×1.1 (중첩)
  - "위기관리": 리스크 변동폭 영구 -10%
  - "제국의 유산": 시작 자본 ×2
```

## 8. 밸런스 수치 (초안)

### 시작 값
- 초기 자본: $100
- 초기 섹터: 3개 (부동산, 주식, 사업)

### 목표 진행 속도
- 1K 도달: ~5분
- 10K 도달: ~20분
- 100K 도달: ~1시간
- 1M 도달: ~3시간
- 10M (첫 환생): ~8시간

### 수익률 (자본 대비 %/초)
- 초반: 2~8%/s → 빠른 성장감
- 중반: 수익률은 유지, 비용 스케일링으로 체감 속도 조절
- 후반: 이벤트 + 팩터 최적화가 성장의 핵심

---

## 9. 차별화 포인트

1. **슬라이더 = 전략**: 단순 구매가 아닌 비율 조절이 핵심
2. **이벤트 = 판단**: 선택에 따른 리스크/리턴 트레이드오프
3. **팩터 중첩 = 깊이**: 시너지 조합 찾기
4. **방치 + 개입**: 슬라이더 놔둬도 돌아가지만, 이벤트 대응 시 훨씬 효율적
